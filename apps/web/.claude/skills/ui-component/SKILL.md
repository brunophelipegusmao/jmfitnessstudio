# Componente UI

## Visão geral

Parte de [StyleSeed](https://github.com/bitjaru/styleseed@), esta habilidade gera componentes que respondem a linguagem de design da semente de Toss em vez de improvisar a marcação e o estilo ad hoc. Ele enfatiza tokens semânticos, digitalização previsível, variantes reutilizáveis e almofadas de acessibilidade amigável a dispositivos móveis.

## Como Usar
- Use quando você precisa de um novo componente de UI primordial ou composto dentro de um projeto baseado em StyleSeed
- Usar quando você deseja um componente para corresponder às convenções de sêmen de lançamento existente
- Usar quando um componente deve ser reutilizável, digitalizado e orientado para design
- Usar quando a IA pode inventar espaço, cores ou páginas de interação

## Como Funcionana

### Passo 1: Leia o Contexto de Desenho Local

Antes de gerar código, inspecione a fonte da verdade da sêmen:
- @ @ CODE0@@ para convenções
- @ @ CODE0@@ para tokens semânticos
- pelo menos um componente representativo de @ @ CODE0@ @

Se o usuário já tiver um exemplar local melhor, siga a base de código local sobre um modelo genético.

### Passo 2: Escolha a casa correta

Colocar a palavra onde pertence:
- @ CODE0@@ para princípios e blocos de baixo nível
- @ @ CODE0@@ para seções compostas ou almofadas multi- partes

Não crie um novo princípio se um existente puder ser estendido com segurança.

### Passo 3: Siga as regras estruturais

Use estes cartões a menos que o projeto host discorde forte:
- declaração de função em vez de @ @ CODE0@@ component
- @ @ CODE0@@ or equivalente native prop tiping
- @ @ CODE0@@ passthrough support
- `cn()` ou a fusão de classe padrão do projeto
- @ @ CODE0@@ for component identification
- AVE ou equivalente apenas quando como variantes são realmente necessárias

### Passo 4: Use apenas os Tokens Semânticos

Não codifique os valores visíveis se o sistema de design tiver um token para eles.

Exemplos preferenciais:
- @ @ CODE0 @ @
- @ @ CODE0 @ @
- @ @ CODE0 @ @
- @ @ CODE0 @ @
- @ @ CODE0 @ @

### Passo 5: Preservar a tipografia e espaço de sêmens de estilo

- Usar a escada já definida pela semente
- Preferer os músculos de 6px
- Usar utilitários de espaço gráfico onde apoiados
- Manter o texto de exibição e cabeçalho apertado, texto corporal legível, legendas contas

### Passo 6: Asse em Acessibilidade

- Alvos de tocar devem ser pelo menos 44x44px para elementos interativos
- O foco do teclado deve estar visível
- Passar por @ @ CODE0@@@ atributos quando apropriado
- Respeito preferências de movimento reduzido para movimento não essencial

## Melhores Práticas

- Compositor de princípios existentes antes de inventar novos
- Mandenha a API do componente pequena e previsível
- Prefere classes de layout semântico sobre valores arbitrarios
- Exportar componentes nomeados a menos que o projeto host use outro bloco consistentemente
